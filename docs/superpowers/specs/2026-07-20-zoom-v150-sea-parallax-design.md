# ズーム演出の詰め v1.50（構え位置・段階倍率・海パララックス） Design

日付: 2026-07-20
前提: v1.49 のズーム演出（`docs/superpowers/plans/2026-07-20-zoom-enemy-scale.md`）に対する
実機フィードバック2件への対応。

## 1. 背景（実機FBと原因）

- FB①「10窓にズームしても大きくなった感じがしない＋数直線を少し隠す」
  - 原因1: 対数カーブが早く飽和し、100窓→10窓の伸びが1.2倍しかない。現行3パラメータ
    （FULL_SCALE/MAX_SCALE/CURVE）では「100窓はそのまま・10窓だけ大きく」が事実上できない
  - 原因2: 敵は水面（画面高さ55%）に足を置いて上へ育つが、数直線が35%にあり、
    間の余白が画面高さの20%しかない。10窓のドローンは既に入りきっていない
- FB②「海がそのままだからズームに見えない」

数字いじりでは両立できないため、作りを3点セットで変更する（2026-07-20 ぽこぴぃ承認済み。
FB①②はセットで設計する——「寄った感」は敵の大きさ×背景の動きの掛け算で決まるため）。

## 2. 設計の全体像

3つの変更をすべて「ズームレベル」（`zoomLevel = log10(spanRatio)`、全体=0・100窓=1・10窓=2、
中間は連続値）に対する**レベル別テーブル＋線形補間**で表現する。数直線のズームには既に
補間アニメ（game.js の zoomDisp）があり、renderer に渡る `state.zoomMin/zoomMax` は補間中の
表示値なので、テーブル補間にするだけで敵の大きさ・立ち位置・海の寄りが全部同じアニメに乗って
滑らかに動く。追加のアニメ機構は作らない。

### ① 敵の立ち位置（足元を下寄りに）

- 現行: `baseCenterY = 0.55H − shipH/2`（足元が常に水平線 0.55H）
- 変更: 足元の高さをズームレベル別テーブル `ANCHOR_BY_LEVEL: [0.55, 0.62, 0.70]`
  （画面高さに対する割合）で決め、中間は線形補間。近づくほど敵が画面手前（下）に構える遠近法
- 空とぶ敵の `airLift`（画面高さ割合）はそのまま足元基準に加算（変更なし）
- 安全装置: 敵の上端（centerY − shipH/2）が `数直線Y + TOP_MARGIN` より上に出る場合は、
  上端がそこに収まるまで centerY を下へ押し下げる（数直線を隠さない保証）。
  過去FB「大きい船が目盛りを隠す」の再発防止はこのキャップが担う
- ズームが起きない場面（zoomable=false: FIRE/RESULT・みならいの測量）は従来どおり
  `STATIC_ANCHOR: 0.55` 固定。撃沈アニメの位置は1pxも変わらない

### ② 敵の倍率（段階ごとの直接指定）

- 現行の `FULL_SCALE / MAX_SCALE / CURVE`（対数カーブ）を廃止し、
  `BY_LEVEL: [0.45, 2.0, 3.4]`（全体／100窓／10窓）の直接指定＋線形補間に置き換える
- 初期値の意図: 全体・100窓は v1.49 とほぼ同じ、10窓だけ約1.4倍に伸ばす。
  ①で上方向の余白が約2倍になるため 3.4 でも数直線に届かない
- `STATIC_SCALE: 1.0` と `isZoomableScene()` は変更しない
- zoomLevel が 2 を超える場合（将来の1窓ズーム等）はテーブル末尾でクランプ

### ③ 海のパララックス

- seaView の背景描画（renderer.js の `drawImage(bgImg, …)`）を、ズームレベル別の拡大＋
  横パンつきのソース矩形切り出しに変える
- 拡大: `SCALE_BY_LEVEL: [1.0, 1.15, 1.3]`＋線形補間。**水平線（画像高さ53%）を不動点**に
  拡大する＝canvas 上の水平線 0.552H は動かない。敵の足元・しぶき・島の位置関係は崩れない
- 横パン: 敵の画面上の位置（enemyX）が中央からずれている割合 × `PAN_FACTOR: 0.15` だけ、
  背景のソース中心を同方向にずらす。ズーム補間中に敵が横へ滑ると海も同方向に流れ、
  「カメラが敵に寄っていく」ように見える
- ソース矩形は画像の端を超えないようクランプ（端の黒帯・引き伸ばしを出さない）
- zoomable=false の場面は scale=1.0・pan=0（従来の全面描画と同じ）。発射で全体ビューに
  戻る瞬間は、既存のズームアウトアニメに乗って海も一緒に引く
- 夕方（sea-evening）・夜（sea-night）も同じ処理。3枚とも水平線53%・crop 0.96 の
  共通仕様を前提とする（現行コードも共通処理）。実機確認でずれが見えたら画像別の
  horizon 定数に拡張する
- 島（island-cutout）・砲台・数直線など自陣の要素は拡大もパンもしない
  （カメラは「敵に寄る」のであって自陣は手前にあるまま、という整理）

## 3. 部品の切り分け

計算はすべて `js/camera.js` の純関数に置き、renderer は戻り値を使うだけにする。

- `zoomLevelOf(zoomMin, zoomMax, CONFIG)` — 連続ズームレベル（0〜2、クランプ）。内部共通部品
- `lerpByLevel(table, level)` — レベル別テーブルの線形補間。内部共通部品
- `enemyCamScale(zoomMin, zoomMax, CONFIG, zoomable)` — シグネチャ不変。中身を BY_LEVEL 補間に
- `enemyAnchorFrac(zoomMin, zoomMax, CONFIG, zoomable)` — 足元の高さ割合（新規）
- `seaCamera(zoomMin, zoomMax, enemyXFrac, CONFIG, zoomable)` — `{scale, panFrac}`（新規）
- `isZoomableScene(phase, stage)` — 変更なし

### config.js（調整はここだけ）

```js
ZOOM_ENEMY: {
  BY_LEVEL: [0.45, 2.0, 3.4],       // 敵の倍率: [全体, 100窓, 10窓]
  ANCHOR_BY_LEVEL: [0.55, 0.62, 0.70], // 足元の高さ(画面割合): 同上
  STATIC_SCALE: 1.0,                 // ズームが起きない場面の倍率（従来どおり）
  STATIC_ANCHOR: 0.55,               // 同・足元（水平線）
  TOP_MARGIN: 16,                    // 敵の上端と数直線の最低すき間px（安全装置）
},
ZOOM_SEA: {
  SCALE_BY_LEVEL: [1.0, 1.15, 1.3], // 海の拡大率: [全体, 100窓, 10窓]
  PAN_FACTOR: 0.15,                  // 敵の横ずれに対する海の流れ量
},
```

## 4. 変更しないもの（制約の再確認）

- 内部の数直線スケール 0〜1000、測量窓・命中判定・スコアのルール
- FIRE/RESULT とみならいの測量は常に等倍・従来配置（isZoomableScene の設計は蒸し返さない）
- 既存テストを1本も壊さない（`npm test` 緑のまま）

## 5. テスト方針

`tests/camera.test.js` に追加:

- BY_LEVEL 補間: 全体/100窓/10窓でテーブル値どおり・中間spanで単調増加・範囲外クランプ
- anchor: 同上＋zoomable=false で STATIC_ANCHOR
- seaCamera: レベル別 scale・敵中央で pan=0・敵が右寄りなら pan>0・zoomable=false で {1, 0}
- 後方互換: zoomable=false の enemyCamScale が STATIC_SCALE を返す（既存テストの維持）

renderer の描画（安全キャップ・ソース矩形クランプ）は実機のブラウザ検証で確認する。
