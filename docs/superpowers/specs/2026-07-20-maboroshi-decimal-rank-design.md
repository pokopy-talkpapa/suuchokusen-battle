# ミクロの海（小数の数直線）＋ズーム演出大胆化 設計書

日付: 2026-07-20
状態: ぽこぴぃ承認済みの壁打ち内容を清書したもの
前提バージョン: v1.48（実機FB修正完了・テスト98本合格・公開済み）

## 0. 一言でいうと

でんせつの次に4つ目のランク**「まぼろしの砲手」**を追加する。
0〜10の「ミクロの海」で、0.1刻みの小数（例: 3.4）を読んで撃つ。
あわせて「ズームすると敵が目に見えて大きくなる」演出を作り、既存ランクにも還元する。

## 1. 決定事項（壁打ちの結論）

| 論点 | 決定 | 却下した案 |
|---|---|---|
| 小数の深さ | **0〜10の海で0.1まで**（正解例: 3.4） | 0.01まで／0〜100で34.7 → 5つ目以降の候補として保留 |
| ズーム演出強化 | **小数ランクと一緒に作り、既存ランク（特にでんせつ）にも効かせる** | 小数だけ／先に単独で直す |
| 世界観 | **「1と1のあいだ」に広がるミクロの海に潜る** | 同じ海の超遠距離（特別感がない） |
| プレイヤー側の見た目 | **島・砲台もミクロ用に丸ごと差し替え** | 海と敵だけ／色味フィルタ |
| 解放条件 | **でんせつで9連続命中**（3→6→9の等差） | 10連続／累計方式 |
| ランク名 | **まぼろしの砲手** | ミクロ砲手／しずくの砲手 |
| 親向けレポート | 優先順位を下げて今回はやらない（数直線は親の悩みとして顕在化しにくいため） | — |

## 2. あそびの中身

### 2.1 ランク構造

- ランクは4段になる: みならい → いっちょまえ → でんせつ → **まぼろし**
- でんせつ（maxLevel 3）で9連続命中すると maxLevel 4 に昇格
- 下位ランクでの自由練習は従来どおり（カウント対象は最上位ランクのみ＝養殖防止ルール維持）

### 2.2 まぼろしランクの数直線

- 表示は **0〜10**。正解は0.1刻み（例: 3.4）
- 遊びの構造は「いっちょまえ」と相似形:
  全体を見て「3と4のあいだだ」→ 窓を覗いて「3.4だ」と読む
- 測量: 0〜10全体（1目盛り）→ ターゲットを含む「1の窓」（0.1目盛り）
- 照準: 1目盛りのままズーム可能で0.1目盛りの窓（でんせつのaim構造の流用）
- **内部実装は現行の0〜1000スケールをそのまま使い、表示だけ÷100して小数にする**
  （例: 内部340 → 表示3.4。物理・判定・ズームのロジックは無傷）

### 2.3 入力

- 「よんでうつ」モードのテンキーに**小数点ボタン**を追加（3.4と打てる）
- メモ表示・結果表示も小数表記に対応

### 2.4 難易度

- hitMargin・enemyScale 等は STAGES の1か所で調整できる作りを維持
- 初期値は仮置きし、実機でぽこぴぃが微調整する前提

## 3. ズーム演出の大胆化（共通機構）

- **カメラの倍率に敵の描画サイズを連動させる**仕組みを新設
- 全体ビュー: 敵は「ちっぽけなシルエット」（今より大胆に小さく）
- 窓を覗く: 敵がぐんと大きくなり「目盛りの間にいる！」が目でわかる
- この機構は**でんせつランクにもそのまま適用**する（いっちょまえへの適用は実機の見え方で判断）
- 倍率カーブ・最小/最大サイズは実機調整前提の定数にする

## 4. 見た目と素材

世界観: 望遠鏡の先、「1と1のあいだ」に広がる小さな海。
小数の本質（数の間にまだ世界がある）がそのまま景色になる。

新規素材（Codex画像生成・既存の絵柄に忠実に寄せる）:

| 素材 | 内容 | 枚数 |
|---|---|---|
| 海背景 | ミクロの海（キラキラの水面・大きな水滴や葉っぱなどスケール感の狂った世界） | 1 |
| 敵 | 小さい敵。候補: アメンボ型メカ／豆潜水艦／妖精船 → 生成して見比べて決定 | 1 |
| 敵の撃沈 | 3コマ（既存 sink-1/2/3 方式） | 3 |
| 島 | ミクロ用（候補: 貝がらの砦） | 1 |
| 砲台 | ミクロ用（候補: 豆大砲） | 1 |

- 島・砲台をランクごとに切り替える仕組みを追加する（敵スプライト方式の流用）
- 既存3ランクの見た目は変えない（ズーム演出の変化を除く）

## 5. 裏の変更点（実装レベルの要点）

- `unlock.js`: maxLevel の上限 3→4（clamp・保存データ読み込みも4対応）
- `stage.js`: nextRankNeed に「でんせつ→まぼろし: 9連続」を追加
- `config.js`: STAGES に4つ目のエントリ追加・UNLOCK に新しきい値・島/砲台のランク別指定
- 数直線ラベルの**小数表示変換層**を1枚かませる（ruler/renderer/aim の表示系のみ）
- `numpad.js`: 小数点キー追加（まぼろしランクのみ表示）
- ランクチップUI・昇格演出・チュートリアル文言の4ランク対応
- テスト: 既存98本を守りつつ、小数表示変換・小数点入力・ランク4解放・ズーム連動スケールの分を追加

## 6. 作る順番（フェーズ分け）

1. **ズーム演出の大胆化** — 既存ランクで目に見えて良くなる。単独でも価値がある
2. **ランク4の骨格** — 解放条件・小数表示・テンキー小数点（仮素材で動くところまで）
3. **ミクロ世界の素材** — 背景・敵・島砲台の生成と組み込み
4. **実機調整** — 敵サイズ・許容誤差・チュートリアル文言

## 7. やらないこと（今回のスコープ外）

- 0.01の深さ・0〜100の小数（5つ目以降の候補として保留）
- 親向けミニレポート（優先順位下げ）
- 対戦・図鑑・PWA化・週間チャレンジ（別構想として保留）
- 既存3ランクの遊びのルール変更（ズーム演出以外は触らない）
